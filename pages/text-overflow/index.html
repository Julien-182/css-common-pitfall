Text overflow

Here is a user list from your application.

The design of the application requires cards to have 200px of width. But when using theses cards in real use case, you realize that data can be a lot larger than expected.

For exemple, the 2nd card user name wrap and go to a new line. Even worse, the 3rd card text goes outside the component ! Let's fix this mess.

We can see that the attempt to limit the text size with max-width does not work.

Here are the magic properties to fix text-overflow.

  • white-space: nowrap;

    The white-space property control how space between characters is handled. The value nowrap tells the browser to suppress white-space and line break. We fixed unwanted line breaks.

  • overflow: hidden;

    The overflow property control what to do with content not fitting in the parent element. The hidden value allow us to hide text initially going outside the card.

  • text-overflow: ellipsis;

    The text-overflow property manage how to display hidden overflow text content. The value ellipsis makes a more natural look and make the user understand that more text exists outside what is displayed.

Now that our design is fixed, I advise to add a title attribute on the text DOM for the sake of accessibility. On hover, users can still see the complete data in a tooltip, managed by your browser. Hover the cards to see.
