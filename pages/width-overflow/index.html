Width overflow

Here is the situation:

You want to have a full width frame with a border to put some content in. You set it a width of 100% but you notice that an horizontal scrollbar appear at the bottom of the screen.

BUT WHY ???

The issue here is that, by default, the width of an element (here <div>) is based on the content size.

So both margin and border widths are added to our 100% width. You can try add bigger borders and margins, you'll see that the scrollbar keep growing !

So how do I get rid of it ?

Two solution available :

  • The first one is simple math. You need to take account of the borders and margin (left and right) size in your width property. So 100% minus the border size.

    CSS offer the calc() function to calculate values with simple operation and different units

    Try the folowing code in the editor : width: calc(100% - (2 * 2 * 1em)); (2 times 2 border/padding of 1em) and see the scrollbar disappear.

  • The other solution is more elegant and maintainable : the box-sizing property.

    In the first part, I mentioned that by default, border and margin sizes were not taken into account into the width definition. It is the default value of the property: content-box

    BUT, you can change this default, by setting border-box value instead. In this mode, padin, borders, and margin size will be included in element width. Try box-sizing: border-box with a width of 100%

Now you can say goodbye to unwanted scrollbars ;)
